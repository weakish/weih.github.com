Who am I?

卫昊，89年生，目前居住在北京，在一家通信公司做协议分析兼程序员的工作。业余时间是个Web开发者，使用Mac、Emacs等工具开发Rails程序，关注Rails相关的技术，并且希望可以成为一位全职的Rails开发人员。

What can I do?

Rails & Ruby
我接触Rails以及Ruby的时间并不长，大概只有大半年的时间。之前接触过Python，在读过《Ruby编程语言》一书后，转型为Ruby程序员。Rails方面，比较大众化的书和Web上的资料都看过，写过几个小应用，熟悉Rails的基本开发套件（RSpec、Guard等）的使用，可以独立开发业务逻辑不是非常复杂的站点。Ruby方面，我还没有写过Gem，不过我有很特意的去单独学习Ruby语言本身，了解基本的Ruby元编程技术，以及代码编写规范。
HTML & CSS
我想作为Rails程序员，基本上都是全能的，从前到后都得会。虽然不像前端人员那样对CSS的各个细节都了解的那么清楚，但也可以独立一人完成站点的设计。我了解HTML5及CSS3中比较重要的部分，知道一点Web开发中前端常用的Hack（如在不支持CSS3的浏览器中实现背景伪渐变等）。
Javascript
Javascript算是我的软肋，目前也仅停留在使用的阶段，主要还是用jQuery操作DOM。我没有很刻意的去学Javascript，不过我知道Javascript的面向原型的模式，以及基本的语法。
Ajax
Ajax作为Web2.0中极其重要的一环不会是不行的。了解Ajax的概念，知道如何在Rails中使用Ajax，知道实现Ajax的几种方式（JS，json，jsonp等）。
Database
做后端当然离不开数据库，我不是DBA，我也算是个SQL的门外汉，不过借助ORM还是能应付得来。关于NoSql方面，我写的第一个应用就是Mongodb，我知道NoSQL跟RDBMS的区别。在做第一个应用时，我使用了Mongodb，但是我是觉得多少有些大材小用，也并没有体会到NoSQL数据库相对MySQL的优势。我想跟个人经验不足以及项目不够大都有关系。
Git
我个人所写的项目都托管在Github，我使用过SVN，所以我能很好的理解分布式版本控制相对集中式版本控制的优势以及理念。我向开源项目发过pull request，了解采用Git以及Github的项目的整体开发流程。
Deploy
自己写过小的Rails应用，也有自己的VPS，知道基本的服务端部署，如Nginx，Linux服务器管理等。
Design
我不是设计人员，不过我很喜欢设计，偶尔也会看看设计方面的书籍，博客、学学Photoshop等。个人觉得UX从某种意义上来说比后端更重要，也是最容易被忽视的地方（尤其是国内的网站）。
TCP/IP
我做过TCP/IP协议栈的编写工作，所以比一般的Web开发人员在这一方面要懂得多一些，同样的还有HTTP协议。
Others
英语六级，我看过很多的英文书籍、资料，在技术阅读上没有什么问题，听力优于口语。自学过日语，能看懂简单的日文，听力优于口语（个人认为我的日语听力要优于英语）。
